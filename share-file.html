share-file

Publish any file to a shareable URL backed by a secret GitHub gist. Anyone with the link can view it.

Uses existing GitHub auth Single bash script Works from terminal or agent Updates reuse the same URL

Get started

Install

mkdir -p ~/.local/bin && \
  curl -fsSL https://raw.githubusercontent.com/alecgard/share-file/main/bin/share-file \
    -o ~/.local/bin/share-file && \
  chmod +x ~/.local/bin/share-file

Share a file

$ share-file dashboard.html
Source:   https://gist.github.com/you/abc123
Rendered: https://alecgard.github.io/share-file/?abc123
(copied to clipboard)

That rendered URL works for anyone you send it to. Here's a live example.

Use it from an agent

Install the agent skill so Claude Code (or any agent that loads skills) can call share-file for you:

mkdir -p ~/.claude/skills/share-file && \
  curl -fsSL https://raw.githubusercontent.com/alecgard/share-file/main/skill/SKILL.md \
    -o ~/.claude/skills/share-file/SKILL.md

Then ask things like "share this dashboard with my team" or "give me a link to that screenshot" — it calls share-file --json and replies with the rendered URL.

For project-scoped install, drop it in .claude/skills/share-file/ inside the repo instead.

Common operations

Update an existing share

The URL stays the same.

share-file --update abc123 dashboard.html

Read from stdin

echo "$html" | share-file --stdin --filename report.html

--filename is required so the viewer can detect the MIME type.

JSON output for scripts and agents

$ share-file --json /path/to/chart.png
{
  "gist_id": "abc123",
  "source_url": "https://gist.github.com/you/abc123",
  "rendered_url": "https://alecgard.github.io/share-file/?abc123",
  "filename": "chart.png",
  "mime_type": "image/png",
  "encoding": "base64"
}

Public gist instead of secret

share-file --public --desc "Q3 results" dashboard.html

Supported file types

HTMLSandboxed iframe
ImagesPNG, JPG, GIF, SVG, WebP
DocumentsPDF, Markdown, plain text
MediaAudio and video
CodeJSON, XML, source files
Anything elseDownload link

Size limit: 900 KB encoded payload (~675 KB raw for binaries).

Self-host your own viewer

By default shares render through the public viewer at alecgard.github.io/share-file/. You might want your own for a dedicated rate-limit bucket, your own domain, or independence from the public viewer's uptime.

Gists created against either viewer work in both — the gist ID travels.

gh repo clone alecgard/share-file
cd share-file
./bin/setup

Re-running is safe. After ~30s your viewer is live at https://<you>.github.io/share-file/.

How the script picks a viewer

  1. $SHARE_FILE_VIEWER env var (overrides everything)
  2. ~/.config/share-file/viewer (written by bin/setup)
  3. Public default (https://alecgard.github.io/share-file/)

Limits and caveats

Gist URLs are unguessable but not access-controlled. Don't share sensitive content.
  • Unguessable, not authenticated. Anyone with the gist ID can view.
  • Rate limit. Unauthenticated GitHub API allows 60 req/hr per viewer IP. Self-hosting gives you your own bucket.
  • Single file per share. Multi-file artifacts: inline assets or use a CDN.
